# 不足与改进方向（论文正文，无小标题）

本系统在实现既定功能的同时仍存在若干不足。在模型与效果方面，分类与生成质量高度依赖所选本地模型的规模与预训练语料，小参数模型对边界样本、长正文及多意图混合邮件易出现误判或回复不够贴切、润色风格偏移等问题；当前分类为二分类且未区分邮件类型与优先级，回复与润色也未针对不同语种或领域做差异化提示设计。在评估与验证方面，受研究范围与资源所限，未引入标准邮件数据集进行准确率、精确率、召回率与 F1 的定量测评，也未对回复与润色结果进行人工打分或与参考回复的相似度分析，与云端大模型 API 或其它基线的对比实验缺失，现有结论主要基于功能正确性与实际使用观察。在功能与鲁棒性方面，邮箱服务器配置仅覆盖部分常见服务商，未支持 OAuth2 等现代授权方式；邮件解析对复杂 MIME 结构与嵌套 HTML 的鲁棒性有限，极端情况下可能出现正文或附件提取不完整；缓存采用 FIFO 淘汰且未区分任务类型，在混合使用分类、回复与润色时可能提前淘汰仍有价值的条目。在安全与部署方面，账号密码或授权码以明文形式持久化于用户目录，存在泄露风险；系统目前主要在 Windows 与单机环境下测试，对 Linux、macOS 及多用户、高并发场景的适配与验证不足。

针对上述不足，改进方向可归纳为以下几类。在效果与评估上，引入公开或自建的标注邮件数据集，对垃圾邮件分类进行准确率、精确率、召回率与 F1 的定量评估，并建立回复与润色的人工评价或自动指标（如与参考回复的语义相似度），与云端 API 或不同规模本地模型进行对比实验，为模型选择与提示优化提供依据；在提示工程中增加少样本示例或按语种、邮件类型拆分的提示模板，有条件的可对本地模型进行轻量级微调或知识蒸馏，在保持离线部署的前提下提升分类与生成质量。在功能与鲁棒性上，扩展邮箱服务器配置以覆盖更多服务商并探索 OAuth2 集成，增强邮件解析对复杂 MIME 与 HTML 的处理能力并完善异常与回退逻辑，对缓存策略引入 LRU 或按任务类型与访问频率的差异化淘汰以提升命中率。在安全与部署上，对持久化账号信息采用系统密钥环或加密存储，避免明文落盘；在更多操作系统与运行环境下进行兼容性测试，并根据实际需求考虑多用户隔离、并发控制与日志审计等能力，使系统在满足个人与轻量办公场景的基础上具备更强的可推广性。
