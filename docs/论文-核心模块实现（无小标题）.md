# 核心模块实现（论文正文，无小标题）

本系统核心模块围绕本地语言模型推理、邮件接入与解析、账号配置、业务逻辑及表示层展开，各模块通过统一数据结构与接口契约协作，完成邮件拉取、分类、回复生成与润色及发送的完整流程。

本地大模型封装采用单例模式与双后端抽象，以降低模型重复加载带来的内存与延迟开销。该模块在类级别维护唯一实例，并通过线程锁保证并发下仅执行一次加载；在首次调用统一生成接口时才完成模型与分词器的加载，实现延迟加载。双后端指在进程内通过 Transformers 从本地目录加载因果语言模型与分词器，或通过 HTTP 调用本地 Ollama 的生成接口；前者采用 local_files_only 实现完全离线并按设备选择 CPU 或 CUDA，后者仅透传 prompt 与温度、生成长度等参数。对外统一提供 generate(prompt, temperature, max_tokens, use_cache) 接口，内部按当前后端分发并对解码结果做统一清洗，使上层业务与具体推理实现解耦。邮件客户端基于 IMAP 与 SMTP 实现收信与发信，登录后可发送 IMAP ID 命令以通过部分服务商的安全检测；拉取时先对指定文件夹 SELECT 并校验状态再 SEARCH，按 UID 倒序取最近若干封的原始数据交由解析模块，若失败则将服务器返回信息解码后纳入异常便于诊断。发信以 MIME 多部件封装正文与可选附件，按账号配置选用 SMTP 的 SSL 或 STARTTLS。邮件解析将单封邮件的原始字节流转为结构化数据：对发件人、主题等头字段做 MIME 解码，正文优先提取 text/plain 否则由 text/html 经正则去标签得纯文本，并提取附件与正文内链接写入单封邮件结构。账号存储在内存中维护多账号列表与当前选中项，以可重入锁保证线程安全，并将账号信息以 JSON 持久化至用户目录实现跨会话记忆；邮箱服务器配置按邮箱域名映射预定义的 IMAP/SMTP 地址、端口及 SSL，支持多种常见服务商免手工填写。

业务层在生成接口之上实现分类、回复与润色。分类采用规则预筛与模型判别结合：主题含广告类标记直接判为垃圾，主题与正文极短且无营销关键词时倾向判为正常，其余样本将主题与正文截断后与固定分类提示词拼接，以低温度与小生成长度调用生成并对输出做关键词匹配得到二分类标签，分类结果可写入缓存以减少重复推理。回复生成将邮件主题与正文组织为结构化提示词，约束模型仅输出一至两句直接回答、不重复原文且无前缀与套话，生成后经行级去重与按句末标点分句、去重及长度截断。润色按自然、正式、商务等风格选择说明并与保持原意、篇幅接近等通用规则组成提示词，按原文长度自适应设置生成长度上限，对极短原文仅保留润色结果首句。表示层由 PyQt5 主窗口与添加账号对话框构成，将拉取、分类、生成、润色与发送等耗时操作封装在 QThread 工作线程中执行，通过完成与错误信号回传主线程更新界面或弹出提示，保证界面响应与 GUI 线程安全；模型选择通过扫描项目内目录及环境变量得到可选路径，用户切换时更新环境变量并清空单例以便下次按新后端加载。缓存模块提供容量受限、线程安全的键值存储，对生成调用按提示与参数缓存结果，超容时按 FIFO 淘汰，用于分类与重复请求以降低推理次数与延迟。上述设计通过单例双后端、延迟加载、异步工作线程与配置外置，在实现完整业务流程的同时兼顾资源占用、响应速度与可维护性。
