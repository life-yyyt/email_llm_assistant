# 邮件分类与回复技术

（基于本项目的学术化表述，可用于毕业论文“关键技术”或“实现方法”等章节）

---

## 一、邮件分类技术

### 1.1 任务定义与设计目标

本系统将邮件分类任务形式化为**二分类问题**：在给定邮件主题与正文的前提下，判定该邮件属于「正常邮件」或「垃圾邮件」。设计目标是在保证正常邮件不被误判（低误杀率）的前提下，对明显具有广告、推销、群发营销等特征的邮件进行识别与标记，以便用户优先处理个人与工作往来邮件。

### 1.2 规则预筛与混合决策框架

为兼顾分类准确率与推理效率，采用**规则预筛与神经语言模型判别相结合**的混合决策框架。

**（1）规则预筛——直接判定**

- **显式垃圾标记**：若邮件主题中出现「广告」「推广」或「<广告>」等标记，直接判定为垃圾邮件，不再调用模型。该类标记多为邮件系统或发件方主动添加，可信度高。
- **显式正常倾向**：当主题长度不超过 2 个字符且正文长度小于 200 字符时，若主题与正文中均未出现「促销」「优惠」「点击」「领奖」「注册」「订阅」「退订」「广告」「推广」等营销相关关键词，则倾向判定为正常邮件并直接返回。该规则用于过滤无主题或极短私人问候类邮件，降低小模型对短文本的误判率。

**（2）模型判别——不确定样本**

对未在上述规则中直接判定的邮件，将主题与正文拼接为统一文本，截断至 1500 字符后，与固定**分类提示词（SPAM_PROMPT）** 一起输入本地因果语言模型。提示词明确约定：仅当邮件「明显」具有垃圾特征（如主题或正文带广告、推销、领奖、点击链接、群发营销、新闻头条推送等）时才选择「垃圾邮件」；个人问候、私信、工作学习往来、无主题或简短主题的私人内容一律视为正常邮件；对拿不准、无明显广告推销特征的样本，一律答「正常邮件」。该设计通过**语义约束与负向先验**降低小模型将正常邮件误判为垃圾的概率。

**（3）生成参数与输出解析**

分类任务采用**低温度**（temperature = 0.1）与**极小生成长度**（max_tokens = 16），以稳定输出并缩短解码时间；模型输出经 strip 后，若包含「垃圾邮件」则返回「垃圾邮件」，否则返回「正常邮件」。同时，对分类任务默认启用**缓存**（use_cache = True），对相同或近似输入直接返回历史结果，减少重复推理。

综上，邮件分类技术通过**规则预筛 + 提示词约束 + 低温度小长度生成 + 缓存**，在保证决策可解释性与稳定性的前提下，实现了对垃圾邮件的有效识别与对正常邮件的保护。

## 二、邮件回复技术

### 2.1 任务定义与设计目标

邮件回复技术的任务是：在给定来信主题与正文的条件下，由系统自动生成一封**简短、直接、可发送**的回复内容。设计目标包括：（1）回复内容紧扣来信主题，对问题类来信给出回答、对请求类来信给出确认、对问候类来信给出简短回应；（2）输出简洁，控制在 1～2 句以内，避免冗长与重复；（3）语言与来信主体语言一致，不中英混杂；（4）不包含解释性文字、前缀标签或套话，仅输出可直接作为邮件正文的文本。上述目标旨在适配小参数规模语言模型的能力边界，并满足日常邮件场景下“快速回复”的需求。

### 2.2 提示词设计与生成约束

系统通过**结构化提示词（Prompt）** 将任务与约束显式传递给模型。提示词由以下部分构成：

- **角色与任务**：明确模型扮演「邮件自动回复助手」，任务为「阅读邮件内容并只给出一个非常简短的回复」。
- **规则约束**：  
  - 只围绕原邮件内容做出直接回答（对问题则回答问题，对请求则简单确认会处理，对问候则简短回应）；  
  - 仅使用 1 句或最多 2 句，总长度尽量控制在 40 个汉字或 100 个字符以内；  
  - 输出语言须与原邮件主体语言一致，只使用一种语言，不允许中英混杂；  
  - 不得重复或复述对方原话，不得引用原文内容；  
  - 不得添加解释、分析、多版本列举或序号；  
  - 不得添加问候语或结尾套话，只输出需要发出的核心回复内容。
- **输入格式**：将来信组织为「邮件主题」与「邮件正文」两部分，与上述规则一起构成完整 prompt，末尾以「请直接给出这封邮件的简短回复：」引导模型输出。

生成阶段采用**中等温度**（temperature = 0.6）与**受限生成长度**（max_tokens = 64），在保证一定表达自然度的同时，抑制过长与发散输出。

### 2.3 后处理流水线

小参数语言模型在开放生成时易出现**重复句、多行复述、超出约定长度**等现象。为此，系统在模型输出之后施加**两级后处理**，以得到符合“简短、单意、可发送”要求的最终回复。

**（1）行级去重与格式规整**

对模型输出的多行文本进行扫描，若相邻非空行内容完全相同，则仅保留一行；同时将连续三个及以上空行压缩为至多两个空行，并对首尾空白进行裁剪。该步骤可消除“同一句话重复多行”的典型小模型偏差，并统一输出格式。

**（2）句子级裁剪与长度控制**

- **分句**：将去重后的文本按行合并为一段，再按中英文句末标点（。！？!?. 等）进行分句，得到句子列表。  
- **去重与截断**：在句子层面再次去重（相同句子仅保留首次出现），并按“短回复”策略仅保留前 2 句。  
- **长度截断**：若合并后的文本长度超过设定上限（如 80 字符），则优先在第一个句号、问号或感叹号处截断，得到一句完整句；若无合适标点则在上限处截断并补省略号，避免输出过长或半句。

通过上述流水线，最终输出被严格限制为 1～2 句、无重复、长度受控的邮件正文，可直接填入回复框或经用户微调后发送。

### 2.4 小结

邮件回复技术通过**结构化提示词明确任务与约束**、**适中的温度与受限的 max_tokens 控制生成规模**，以及**行级去重 + 句子级分句/去重/截断**的后处理流水线，在本地小参数模型上实现了“根据来信内容做出简短回答”的自动化回复能力，兼顾了可用性与输出规范性。

## 三、技术路线小结

| 维度         | 邮件分类技术                               | 邮件回复技术                                           |
|--------------|--------------------------------------------|--------------------------------------------------------|
| 任务形式     | 二分类（正常 / 垃圾）                      | 条件文本生成（输入：来信；输出：简短回复正文）         |
| 规则与模型   | 规则预筛 + 固定 SPAM_PROMPT + 低温度生成   | 结构化 Prompt（角色、任务、多条约束）+ 中等温度生成    |
| 生成参数     | temperature=0.1，max_tokens=16，use_cache=True | temperature=0.6，max_tokens=64，use_cache=False   |
| 后处理       | 输出解析（关键词匹配得到标签）             | 行级去重 → 句子级分句/去重/截断 → 长度截断             |
| 设计侧重点   | 降低误杀、稳定标签、减少重复推理           | 简短、直接、无重复、无前缀、语言一致、可发送           |

上述邮件分类与回复技术的实现与优化策略，可作为毕业论文中“邮件分类与回复技术”或“基于本地语言模型的邮件智能处理”等小节的内容依据，按需增删引用与实验数据即可。

## 四、技术局限（可作论文“不足与展望”小节）

**邮件分类技术的局限**：分类结果依赖规则与提示词的设定，对边界样本（如软广、新闻订阅、活动邀请）的划分仍可能不一致；小参数模型对长文本、多主题或隐含营销意图的邮件理解有限，且仅支持二分类，无法细分为更多类别（如工作/个人/订阅等）。

**邮件回复技术的局限**：为控制输出长度与格式，系统强制将回复压缩为 1～2 句，对需要展开说明或分点答复的邮件场景支持不足；小模型在开放生成时仍可能出现轻微重复、偏离主题或语气生硬，后处理仅能缓解而无法根本消除；回复质量与所选模型（如 Llama、Qwen）及温度等参数强相关，不同环境下表现会有差异。

**邮件润色技术的局限**：润色效果受限于本地小模型的表达与风格迁移能力，“自然/正式/商务”三种风格的区分度在极短文本上可能不明显；对较长正文进行润色时，推理耗时随 max_tokens 增加而上升，在 CPU 或低算力环境下体验偏慢；且系统不提供逐句对照或修改说明，用户无法直接看到“改了什么”。

**系统整体局限**：当前实现面向单语或主语言明确的邮件，对中英混杂、多语交替的邮件未做专门优化；模型与邮件数据均限于本地，不具备联网检索、知识更新或基于用户历史反馈的持续优化能力。
