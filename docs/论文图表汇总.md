## 表

### 表3-1 系统功能需求分析

| 功能模块 | 需求说明 |
|----------|----------|
| 邮件接收 | 能够连接邮箱服务器并获取收件箱中的邮件 |
| 邮件解析 | 能够提取主题、正文、附件和链接等信息 |
| 邮件分类 | 能够将邮件划分为正常邮件和垃圾邮件 |
| 自动回复 | 能够根据邮件内容生成回复草稿 |
| 邮件润色 | 能够对回复内容进行自然、正式、商务风格优化 |
| 邮件发送 | 能够通过 SMTP 协议发送处理后的邮件 |
| 多账号管理 | 能够支持多个邮箱账号的添加、切换与删除 |

### 表3-2 系统功能板块说明

| 功能板块 | 主要作用 |
|----------|----------|
| 邮箱账号管理板块 | 管理邮箱账号及服务器配置信息 |
| 邮件接入与解析板块 | 获取邮件并完成结构化解析 |
| 邮件分类板块 | 判断邮件属于正常邮件或垃圾邮件 |
| 自动回复板块 | 生成适合当前场景的回复草稿 |
| 邮件润色板块 | 对文本进行自然、正式和商务风格优化 |
| 图形界面交互板块 | 提供图形化操作入口和结果展示 |

### 表3-3 主要数据实体设计

| 实体名称 | 关键字段 | 说明 |
|----------|----------|------|
| Account | 邮箱地址、授权码、IMAP/SMTP服务器 | 保存邮箱账号配置 |
| Mail | 主题、发件人、时间、正文、分类结果 | 表示邮件主体信息 |
| Attachment | 附件名称、附件路径 | 表示邮件中的附件信息 |
| Link | 链接地址 | 表示邮件正文中提取出的链接 |

| 链接提取 | 识别正文中的链接地址 |
| 数据组织 | 封装为统一的邮件对象 |
